## 3. 地図見開き（P02–03）

### 目的
全8回を一望し、「今どこにいるか」を直感的に把握する。文章を読むページではなく、ナビゲーションページ。

### 情報ブロック

| ブロック | 内容 | 備考 |
|---------|------|------|
| エリア区分 | 5章構成（下表参照）で視覚的に分ける | 各エリアに背景色・帯 |
| 各回ノード | 回番号・日付・タイトル（短縮版）・スタンプ枠 | 8個 |
| 進行方向 | すごろく型蛇行。矢印で進行方向を明示 | |
| スタンプ枠 | 各回ノード内に1枠 | サイズはスタンプ実物に合わせて後日確定 |
| エリア名 | 5章構成で確定（下表） | RPG世界観 |
| イラストスペース | 自作予定。冒険者・地形モチーフ | なくても読める構造を維持する |

### エリア構成（5章・確定）

| 章 | 回 | エリア名 | 内容テーマ |
|----|-----|---------|-----------|
| 1 | S01–02 | ドーナツ村 | はじめての3DCG・スカルプト（ドーナツ） |
| 2 | S03 | キャラメイク平原 | 自力モデリング（キャラ・マテリアル・テクスチャ） |
| 3 | S04–05 | デザイン湿地 | デザイナー養成（ポスター・仕様書・職人との対話） |
| 4 | S06–07 | シーンメイク魔境 | モデリング②・テクスチャペイント・発光演出 |
| 5 | S08 | ドーナツ村 🏆 | 発表・修了・凱旋（円環構造） |

> S08は出発点「ドーナツ村」に戻る円環構造。ノードを地図上でS01-02の近くに配置し、旅の完結を視覚的に表現する。S08ノードはゴールド特別仕様（グロー付き）で区別する。

### レイアウト方針
- 718×643px（見開き全体）のSVGで実装。左ページ（P02）・右ページ（P03）に分割
- ページ中央（x=359）に断ち切り線が入るため、ノードは必ず左右どちらかのページに収まるよう配置する
- 8ノードをすごろく型に蛇行配置。進行ルートは以下：

```
左ページ（P02）              右ページ（P03）
01(左上) ──────────────→ 02(右上) ──→ 03(右上)
                                          ↓ L字
05(右中) ←──────────────── 04(右中)
↓ L字
06(左下) ──→ 07(右下) ──────────────→ 08(中下)🏆
```

- ノード形状：角丸四角（幅114×高60px、角丸8px）
- ノード配置はグリッドから意図的にフレックスにずらし、地図らしい有機的な印象を持たせる
- 道はL字折れの破線（stroke-dasharray）で表現し、終点に矢印を付与
- 背景：古地図風オフホワイト（ラジアルグラデーション＋クロスハッチ＋等高線装飾＋ビネット）
- ノード内テキスト色：クリーム白（`#F5EAD0`）。背景が明るいため暗い塗りのノードで視認性を確保

## 4. 各回シート（S01–S08）

### 媒体
A6両面印刷・1枚。毎回配布し、コアシートに挟んで蓄積する。

### 表面：ミニ参考書

**目的：** その回で「何を学び・何が核で・どこで詰まりやすいか」が一目でわかること。ワークシートでもチェックリストでもない。

### 共通情報ブロック仕様

| ブロック名 | 内容の性格 | 文字量の目安 |
|-----------|-----------|-------------|
| ① Chapter識別 | 「Chapter 1」など回番号 | 1語 |
| ② 回タイトル | その回の短いタイトル | 15字以内 |
| ③ 日付 | 開催日（例：4/5） | |
| ④ 今日のゴール | その回で目指す状態を1文で | 30字以内 |
| ⑤ キーワード | その回の中心概念 2–4語 | タグ・バッジ的な見た目 |
| ⑥ 核（コア） | 「この回で一番大事なこと」を太字1文で | 40字以内 |
| ⑦ 手順の大枠 | 流れを3–5ステップで（羅列ではなく概念として） | 各ステップ20字以内 |
| ⑧ うまくいくコツ | ポジティブな助言 1–2点 | 各30字以内 |
| ⑨ よくある失敗 | 気をつけるポイント 1–2点 | 各30字以内 |
| ⑩ 今回の成果物 | 何ができあがるか | 1文 |
| ⑪ 記録欄 | 「今日の一言メモ」など軽い自由記述 | 2–3行分のスペース |
| ⑫ 次回予告 | 次回タイトルと一言 | 小さく末尾に |

### 文体方針
- 「〜しましょう」ではなく「〜するとうまくいく」「〜を意識しよう」
- 命令・禁止の語調を避け、参考書的・助言的にする
- 専門用語は使うが、初出は簡単な説明を1行添える

### レイアウト方針（表面）
- 95×170mm（バインダー穴込み）に上記ブロックを収める
- 優先順位：①②③ → ④⑤⑥ → ⑦ → ⑧⑨ → ⑩⑫
- ⑦手順は番号付きリストではなく「流れ図」または「横並びステップ」の視覚構造を推奨
- ⑧⑨はアイコン（✓と△など）で視覚的に区別する

### 裏面：記録欄

**目的：** 軽い手書きの痕跡を残す。管理帳票にしない。

| ブロック名 | 内容 | 備考 |
|-----------|------|------|
| ⑪ 今日の一言メモ | 自由記述 | 3–4行分のスペース |
| スタンプ枠 | 受講スタンプ1個分 | 地図ページと連動する場合の予備枠 |
| 次回予告 | 次回タイトルと一言 | 小さく末尾に。表面に収まらない場合はこちらに移動 |

## 5. コアシート：Tips・ルールページ（P04）

### 目的
命名規則・提出方法・よく使うショートカットなど、講座全体を通じて使うルールを一箇所にまとめる。毎回の各回シートに書く必要がなくなり、「困ったらP04を見る」という参照先になる。

### 情報ブロック（案）

| ブロック | 内容 | 備考 |
|---------|------|------|
| ファイル命名規則 | 「名前_回数_内容」など | 第1回でも触れるが、ここが正式な記載場所 |
| 提出方法 | どこに・どうやって出すか | 迷子にならないための手順を簡潔に |
| よく使うショートカット | ツール固有の操作3–5個 | ソフト名を明記。多すぎない |
| 困ったときは | 先生に聞く・保存してから試す 等 | 参考書的な口調で |

> **内容の確定は別途。** 講師側で記載したいルール・Tipsを収集してから詳細設計する。
